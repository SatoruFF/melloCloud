# Мобильное приложение из веб-приложения Mello

## Варианты (от самого быстрого к полноценному)

### 1. **PWA (Progressive Web App)** — уже есть у вас

**Что это:** тот же сайт, но с манифестом и Service Worker. Пользователь может «Установить приложение» в браузере на телефоне и открывать его с главного экрана.

**Плюсы:**
- У вас уже настроен **Vite PWA** (manifest, иконки 192/512). Остаётся доработать офлайн/кэш при необходимости.
- Никакого отдельного кода, один деплой.
- Не нужны App Store / Google Play.
- Бесплатно.

**Минусы:**
- Нет публикации в сторах, ограниченный доступ к нативным API (камера, пуш-уведомления и т.д. — через Web API).
- На iOS ограничения по офлайну и пушам.

**Итог:** самый быстрый вариант — просто продвигать установку PWA (кнопка «Добавить на главный экран» в UI или в документации).

---

### 2. **Capacitor** — лучший вариант для «настоящего» мобильного приложения

**Что это:** обёртка над вашим веб-приложением. Тот же React/Vite-код собирается в статику, а Capacitor упаковывает её в нативные проекты iOS и Android. В сторах публикуются обычные приложения.

**Плюсы:**
- **100% переиспользование кода** — не нужно переписывать на React Native.
- Один кодбаз: веб + iOS + Android (и при желании PWA).
- Доступ к нативным возможностям через плагины (камера, пуш-уведомления, файловая система, статус-бар и т.д.).
- Публикация в App Store и Google Play.
- Бесплатный и открытый (Ionic/Capacitor).

**Минусы:**
- Нужны Xcode (для iOS) и Android Studio, аккаунты разработчика (Apple $99/год, Google — разовый взнос).
- Сборка и подпись приложений сложнее, чем деплой сайта.

**Как подключить (кратко):**
```bash
cd frontend
pnpm add @capacitor/core @capacitor/cli
npx cap init
pnpm build
npx cap add ios
npx cap add android
npx cap sync
```
Дальше открываете `ios/` в Xcode, `android/` в Android Studio и собираете/подписываете как обычные нативные приложения.

**Итог:** если цель — именно приложения в сторах с тем же Mello — **Capacitor самый выгодный вариант**.

---

### 3. **TWA (Trusted Web Activity)** — только Android

**Что это:** приложение в Google Play, которое по сути открывает ваш PWA в полноэкранном Chrome без UI браузера.

**Плюсы:**
- Минимум кода (тонкая обёртка вокруг URL).
- Один кодбаз с веб-версией.
- Бесплатно (кроме взноса в Google Play).

**Минусы:**
- Только Android, не iOS.
- Ограничения как у PWA (нет произвольных нативных модулей).

**Итог:** имеет смысл, если нужен только Android и хочется «приложение в сторе» при минимальных изменениях. Для iOS так не сделаешь.

---

### 4. **React Native** — не рекомендуется для вашего случая

**Что это:** отдельный фреймворк под мобильные приложения. Компоненты не React DOM, а нативные виджеты.

**Минусы для вас:**
- Придётся переписывать UI и навигацию под React Native — **код веб-приложения напрямую не переносится**.
- Два кодбаза (веб + мобильный) или сложная абстракция.
- Дольше и дороже в разработке.

**Итог:** имеет смысл только если вы изначально делаете новое мобильное приложение с нуля и готовы писать под RN.

---

## Рекомендация

| Цель | Вариант |
|------|--------|
| «Чтобы можно было поставить на телефон» быстро и без сторов | **PWA** (у вас уже есть база) |
| Полноценные приложения в App Store и Google Play с тем же кодом | **Capacitor** |
| Только Android в сторе, минимум кода | **TWA** |
| Новое приложение с нуля, максимум нативной производительности | React Native (не для переноса текущего Mello) |

**Оптимальный путь для Mello:**  
Сейчас — доработать и продвигать **PWA** (иконки, манифест уже есть). Когда понадобятся приложения в сторах и нативные фичи — добавить **Capacitor** к тому же фронту и собирать iOS/Android из того же репозитория.
